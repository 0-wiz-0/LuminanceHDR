Features overview

Streamlined localized user interface

Creating HDR files is very easy: you just drop your bracketed shots from file or photos manager to the main window and in few more clicks you have an HDR file which you can then tonemap.

Qtpfsgui is available in the following languages:

  • Czech (outdated)
  • English
  • French (outdated)
  • German
  • Hungarian (outdated)
  • Indonesian (outdated)
  • Italian
  • Polish (outdated)
  • Russian
  • Spanish (outdated)
  • Turkish (outdated)

If your language is not supported or localization is incomplete or outdated and you are willing to take care of that, please learn how to do it.

Supported HDR file formats

Qtpfsgui reads and writes:

  • OpenEXR (.exr)
  • Radiance RGBE (.hdr)
  • Logluv TIFF (.tif)
  • PFS stream (.pfs)

Additionally Raw images can be opened using DCRaw.

Supported LDR file formats

Qtpfsgui supports the following low dynamic range file formats:

  • JPEG
  • TIFF (only 8bpc)
  • PNG (only saving)
  • PBM (only saving)

Qtpfsgui can also write Exif metadata to JPEG, PNG and TIFF files using Exiv2.

Extensive help system

We don't really pretend that HDR imaging is easy. It surely can be easy if lousy results is what you want. For the rest of you we bundle a solid help system to help you getting the most of your pictures.

The help system is, again, localizable. Right now only English version is available, but you can contribute a translation into your native language, if you really want to.
